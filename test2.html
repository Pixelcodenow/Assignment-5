Question-1: What are the different ways to select an element in the DOM?

DOM থেকে কোনো element নির্বাচন করার জন্য কয়েকটি পদ্ধতি আছে।


`getElementById("id")` ব্যবহার করলে নির্দিষ্ট `id` সহ একটি element পাওয়া যায়। `getElementsByClassName("class")` একটি নির্দিষ্ট class এর সব elements এনে দেয়, যা HTMLCollection হিসাবে থাকে।


`getElementsByTagName("tag")` দিয়ে নির্দিষ্ট ট্যাগের সব elements পাওয়া যায়। `querySelector("selector")` CSS সিলেক্টরের মতো কাজ করে এবং প্রথম মিলে যাওয়া element ফেরত দেয়।


`querySelectorAll("selector")` CSS সিলেক্টরের মতো কাজ করে, তবে এটি সব মিল থাকা elements এর একটি NodeList দেয়। এসব পদ্ধতি ব্যবহার করে JavaScript দিয়ে DOM এর element নির্বাচন করা যায়।

Question-2: What is the difference between InnerHTML, InnerText, and textContent?

innerHTML, innerText, এবং textContent তিনটি প্রপার্টি হলেও এদের কাজ ভিন্ন।


innerHTML পুরো HTML কোডসহ element-এর ভেতরের কন্টেন্ট পরিবর্তন বা দেখাতে ব্যবহার হয়। এটি HTML ট্যাগগুলোকেও পার্স করে, তাই যদি এর ভেতরে Bold লেখা থাকে, তাহলে এটি ওয়েবসাইটে bold হয়ে দেখাবে।


innerText শুধুমাত্র দৃশ্যমান (visible) টেক্সট দেখায় এবং HTML ট্যাগগুলোকে ইগনোর করে। এটি স্টাইল করা display: none; থাকা টেক্সট দেখায় না।


textContent পুরো টেক্সট দেখায়, এমনকি যদি সেটি hidden থাকলেও। তবে এটি HTML ট্যাগ পার্স করে না, বরং সরাসরি টেক্সট হিসেবে দেখায়।

Question-3: What is event delegation in the DOM?

Event delegation হলো এমন একটি পদ্ধতি যেখানে parent element-এর উপর ইভেন্ট listener বসিয়ে তার মাধ্যমে child elements-এর ইভেন্ট হ্যান্ডেল করা হয়। সরাসরি প্রতিটি child element-এ ইভেন্ট listener না বসিয়ে, parent element-এ ইভেন্ট ধরে নিয়ে তা নির্দিষ্ট child element-এর জন্য কাজ করানো হয়।


এটি event bubbling এর কারণে সম্ভব হয়, যেখানে ইভেন্ট নিচ থেকে উপরের দিকে propagate করে। যদি অনেক child element থাকে, তাহলে প্রতিটির জন্য আলাদা event listener যোগ করলে performance খারাপ হতে পারে, তাই event delegation ব্যবহার করলে কম কোডে ভালো কাজ করা যায়।


এটি বিশেষভাবে কাজ করে যখন ডায়নামিকভাবে নতুন element যোগ করা হয়, কারণ নতুন element-এ আলাদা করে event listener যোগ না করলেও parent-এর মাধ্যমে সেটি হ্যান্ডেল করা যায়।

Question-4: What is event bubbling in the DOM?

Event bubbling হলো DOM-এর একটি behavior যেখানে একটি child element-এ ইভেন্ট ঘটলে সেটি তার parent, তারপর তার parent-এর parent এভাবে উপরের দিকেও ছড়িয়ে যায়। এটি তখন ঘটে যখন কোনো ইভেন্ট একটি element-এ trigger হয় এবং সেই ইভেন্ট তার ancestor elements গুলোকেও notify করে।


যদি একটি `div` এর ভেতরে `button` থাকে এবং `button`-এ ক্লিক করা হয়, তাহলে প্রথমে সেই `button`-এর ইভেন্ট চলবে, তারপর সেই `div`-এর, তারপর তার parent element-এর ইভেন্ট trigger হবে। এটি ডিফল্টভাবে ঘটে, কারণ ইভেন্ট স্বয়ংক্রিয়ভাবে bubble হয়ে উপরে উঠে।


এই behavior থামানোর জন্য `event.stopPropagation()` ব্যবহার করা হয়। Event bubbling ইভেন্ট হ্যান্ডলিং সহজ করে তোলে এবং একাধিক ইভেন্ট listener না লাগিয়ে parent element-এ ইভেন্ট ধরতে সাহায্য করে।

Question-5: How do you create, add, and remove elements using JavaScript?

JavaScript দিয়ে নতুন element তৈরি করতে document.createElement("tagName") ব্যবহার করা হয়।
নতুন element এর ভিতরে লেখা যোগ করতে innerText বা innerHTML ব্যবহার করা যায়। তৈরি করা element পেজে দেখানোর জন্য appendChild() বা prepend() দিয়ে কোনো parent element এর ভিতরে যোগ করতে হয়। নির্দিষ্ট জায়গায় element বসাতে insertBefore() ব্যবহার করা যায়।

কোনো element মুছে ফেলতে removeChild(childElement) বা সরাসরি remove() ব্যবহার করা হয়। removeChild() প্যারেন্ট থেকে নির্দিষ্ট child element মুছে দেয়। remove() নিজেই নিজেকে পেজ থেকে সরিয়ে ফেলে।

এভাবে JavaScript দিয়ে ডায়নামিকভাবে HTML element তৈরি, যোগ এবং মুছে ফেলা যায়।
